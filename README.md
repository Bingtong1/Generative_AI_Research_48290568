# Generative AI Research: Boon or Bane

A research project investigating the use of Generative AI Tools in academic and professional settings.

## Project Overview

This repository contains all research materials, data, and analysis scripts for the study **"Using Generative AI Tools - Boon or Bane."** The project combines quantitative survey analysis and qualitative interview methods to examine how generative AI tools impact research workflows and outcomes.

## Repository Structure

```
AI_Research/
├── Literature_Review/          # Reference papers, books, and journal articles
├── Quantitative_Analysis/
│   ├── Survey_Data/            # Survey CSVs, Excel files, and questionnaires
│   └── Analysis_Scripts/       # Python scripts for data processing and analysis
├── Qualitative_Analysis/
│   └── Interview_Transcripts/  # Interview recordings and transcripts
├── Drafts_and_Reports/         # Research proposals, conference papers, and final reports
└── Additional_Materials/       # Photos, information sheets, and other media
```

## Data Types

| Folder | Contents | Access |
|---|---|---|
| `Literature_Review/` | Journal articles, books, conference papers | Public |
| `Quantitative_Analysis/` | Survey data (CSV), Python analysis scripts | Public |
| `Qualitative_Analysis/` | Interview transcripts, consent forms, protocols | Restricted |
| `Drafts_and_Reports/` | Research proposals and final papers | Public |
| `Additional_Materials/` | Images and supplementary media | Public |

> **Note:** Qualitative data (interview transcripts and consent forms) may contain personally identifiable information. Access is restricted to authorised team members only.

## How to Navigate

- Start with `Literature_Review/` for background reading and references.
- See `Quantitative_Analysis/Survey_Data/` for raw survey data and `Analysis_Scripts/` to run or modify the analysis.
- Interview data is in `Qualitative_Analysis/Interview_Transcripts/`.
- Final and draft outputs are in `Drafts_and_Reports/`.

## How to Contribute

1. **Fork** this repository to your GitHub account.
2. **Clone** your fork to your local machine.
3. Create a **new branch** with a descriptive name (e.g., `feature/survey-analysis` or `fix/interview-coding`).
4. Make your changes and write a clear **commit message** describing what was changed and why.
5. **Push** your branch and open a **Pull Request** for review.
6. The repository owner will review, request changes if needed, and **merge** approved contributions into `main`.

## Naming Conventions

- Files: use lowercase with underscores, include dates where relevant (e.g., `interview_transcript_eric_2024.docx`)
- Branches: use `feature/`, `fix/`, or `data/` prefixes
- Commit messages: use present tense, describe the change clearly (e.g., `Add cleaned survey data for Q3`)
